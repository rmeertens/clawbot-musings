# World Leaders Sentiment Map - Setup Guide

## Overview
This project fetches tweets from world leaders and analyzes their sentiment in real-time using a local LLM.

## Files
- `world-leaders-sentiment.html` — Interactive map UI
- `fetch_world_leaders_sentiment.py` — Data pipeline (fetch tweets + analyze sentiment)
- `world-map-simplified.json` — GeoJSON map data (auto-generated)
- `world-leaders-data.json` — Sentiment data (generated by pipeline)

## Setup

### 1. Twitter API (Optional)
To fetch live tweets, set up Twitter API v2:

```bash
# Get API credentials from https://developer.twitter.com/
export TWITTER_BEARER_TOKEN="your_bearer_token_here"
```

If not set, the script will use mock data.

### 2. Local LLM Setup
The sentiment analysis runs locally using Ollama:

```bash
# Ensure Ollama is running (should auto-start as a service)
curl http://localhost:11434/api/tags

# Verify the model is available
ollama list | grep qwen3.5
```

If qwen3.5:0.8b-small is not loaded, run:
```bash
ollama pull qwen3.5:0.8b-small
```

### 3. Generate Data
Run the pipeline to fetch tweets and analyze sentiment:

```bash
cd /home/roland/clawbot-musings
python fetch_world_leaders_sentiment.py
```

This creates/updates `world-leaders-data.json`.

### 4. Auto-Update (Cron)
To refresh data hourly, add a cron job:

```bash
0 * * * * cd /home/roland/clawbot-musings && python fetch_world_leaders_sentiment.py >> /tmp/world-leaders.log 2>&1
```

Or use OpenClaw's cron system for scheduled updates.

## Notes

- **Twitter API Rate Limits**: The free tier allows 450 requests/15min. This script fetches ~50 leaders, so it stays well under limits.
- **Sentiment Analysis**: Uses Ollama locally; no external API calls beyond Twitter.
- **Map Data**: GeoJSON sourced from Natural Earth / datasets.org (~14MB, simplified to JSON for faster loading).
- **Performance**: The map renders 190+ countries; SVG paths are optimized for interactivity.

## Troubleshooting

### "TWITTER_BEARER_TOKEN not set"
This is a warning, not an error. The script uses mock sentiment data if the token is missing.

### "http://localhost:11434/api/generate: connection refused"
Ollama is not running. Start it:
```bash
ollama serve &
```

Or check if it's already running as a service:
```bash
systemctl status ollama
```

### Map not rendering
Check browser console (F12) for JSON parsing errors. Verify that `world-map-simplified.json` exists and is valid JSON.

### Sentiment scores are null
The LLM sentiment analysis may have failed. Check:
1. Ollama is running: `curl http://localhost:11434/api/tags`
2. Model is loaded: `ollama list`
3. Recent logs: `tail /tmp/world-leaders.log`

## Extending the Project

### Add more leaders
Edit `WORLD_LEADERS` dict in `fetch_world_leaders_sentiment.py`:
```python
"My Country": ("Leader Name", "twitter_handle"),
```

### Change sentiment model
In `analyze_sentiment_local_llm()`, update the model name:
```python
"model": "llama3.2:3b",  # or any other Ollama model
```

### Customize the map
Edit styles in `world-leaders-sentiment.html`:
- `.sentiment-very-positive`, `.sentiment-positive`, etc.
- Projection: Mercator (simple) → customize `project()` function for other projections
- Tooltip styling: adjust `.country-tooltip`, `.sentiment-bar`
